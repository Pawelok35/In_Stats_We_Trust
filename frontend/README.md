## In Stats We Trust · Frontend

This directory contains the web UI for browsing the pipeline outputs generated by the Python project in the repository root. The app is built with Next.js 16 (App Router), React, TypeScript, Tailwind CSS v4, and [shadcn/ui](https://ui.shadcn.com).

### Project layout

- `src/app` — Next.js routes (`/`, `/weeks/[season]/[week]`, `/reports/[season]/[week]/[slug]`).
- `src/lib/reports.ts` — Server-side helpers that read Markdown reports from `../data/reports/comparisons`.
- `src/components/ui` — shadcn/ui primitives (card, button, badge, etc.).

> **Note**  
> The UI reads Markdown files directly from the repository (`data/reports/comparisons/**/*`). Make sure the Python pipeline has produced reports before starting the frontend.

### Getting started

Install dependencies (already done when scaffolding, run again if needed):

```bash
npm install
```

Run the development server:

```bash
npm run dev
```

Then open [http://localhost:3000](http://localhost:3000) to explore the UI. Changes to files inside `src/` trigger live reloads.

Optional (but recommended): run the FastAPI backend in a separate terminal if you plan to extend the UI with live data.

```bash
uvicorn app.api:app --reload
```

Set the API base URL (defaults to `http://127.0.0.1:8000`) via `.env.local`:

```bash
echo "NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000" > .env.local
```

### Quality checks

```bash
npm run lint   # ESLint
npm run build  # Production build (Next.js + TypeScript)
```

### Adding new UI components

We use [shadcn/ui](https://ui.shadcn.com). Add components via the CLI:

```bash
npx shadcn@latest add <component-name>
```

This keeps theme tokens and generated files consistent with the existing setup.
